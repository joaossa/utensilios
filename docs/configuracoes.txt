5. Variaveis de ambiente realmente relevantes

Frontend:
- obrigatoria:
  - `VITE_API_BASE_URL=https://api.ibgutensilios.com.br`
- regra oficial do Vite:
  - so variaveis com prefixo `VITE_` chegam ao cliente
  - portanto segredo nunca entra em `VITE_*`

API - obrigatorias no codigo atual:
- `DATABASE_URL`
- `JWT_SECRET`
- `CORS_ORIGINS`

API - recomendadas em producao:
- `CORS_ORIGINS=https://ibgutensilios.com.br,https://www.ibgutensilios.com.br`

API - porta:
- o codigo continua usando `process.env.PORT ?? 3001`
- isso significa:
  - `3001` continua sendo fallback local seguro para desenvolvimento
  - se a Hostinger injetar `PORT`, a API ja respeita o valor do ambiente
  - se a conta exigir configuracao manual de `PORT`, a referencia oficial atual da Hostinger puxa para `3000`
- conclusao pratica:
  - em producao, preferir `PORT` entregue pela plataforma
  - se for necessario registrar manualmente no painel por causa de erro real da conta, usar `3000`
  - nao confundir o fallback local `3001` com exigencia de producao da Hostinger

API - usadas apenas em seed controlado:
- `SEED_ADMIN_EMAIL`
- `SEED_ADMIN_PASSWORD`
- `SEED_INCLUDE_SAMPLE_DATA`

Variaveis citadas em documentos antigos, mas nao encontradas no codigo atual:
- `WEB_BASE_URL`
- `RECOVERY_PASSWORD_URL`

Conclusao sobre essas duas variaveis:
- hoje elas nao devem bloquear o deploy
- se uma frente futura de recuperacao de senha for aberta, elas podem voltar a ser necessarias
